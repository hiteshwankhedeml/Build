# Decision Table

* When we need to determine multiple conditions, then we can opt for decision table
* A control will create branch(s) in the business process
* A decision table is a step itself&#x20;
* The control will be used to trigger multiple steps and manage the flow of process at runtime but the decision table is predominantly used to determine the context data at runtime

**Example:**

* If the sales order is OR then approval should go to A
* If the sales order is IN then approval should go to V
* This is an example of dynamic agent determination
