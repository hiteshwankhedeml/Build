# Why we need IAM?

* Identity application management
* All the applications will be accessed through Build Workzone
* In a company there might be 10 developers and 5000 employees
* So onboarding this many users to SAP BTP will be very costly
* For accessing any application like outlook, it goes to login screen of IAM and once authenticated then application can be accessed
* Similarly for BTP the authentication screen comes from IAM
* In our subaccount we need to bind IAM as well as Build
* Then we can configure all the end users as developers
